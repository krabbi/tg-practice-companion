# tg-practice-companion

Персональный Telegram-бот ежедневных практик (КПТ-подход). ТЗ выработано в ходе
структурированного интервью 2026-06-10; неоднозначность требований на момент фиксации — 15%.

> Language policy: этот README — продуктовый документ на русском.
> Issues, код, комментарии, коммиты и PR — на английском (как в tg-smart-inbox).

---

## Цель

Личный Telegram-бот для одного пользователя (психолог, КПТ-подход), который ведёт её
по дневной программе практик: по расписанию присылает напутствия, вопросы для регистрации
мыслей, напоминания о практиках, аудио (гипноз на ночь) и мотивирующие изображения;
записывает все ответы (текст и голос) в долгосрочный дневник с самооценкой «ведёт ли мысль
к моим целям»; каждое утро присылает поддерживающий AI-анализ вчерашнего дня
(«вчера 5 случаев из 10 твои мысли были шикарны — ты на верном пути»). Состав и
периодичность практик меняются еженедельно вручную (одну добавили — другую убрали).

Центральная абстракция — **Практика**: единица контента (вопрос / текст / аудио /
изображение) с периодичностью, расписанием и сроком жизни. Ежечасный вопрос о мыслях,
гипноз на ночь, новая привычка — всё это практики.

## Дневной цикл (эталонная стартовая настройка)

| Время | Событие |
|-------|---------|
| 06:00 | Напутствие на день (из заранее написанного списка, ротация) + утренняя практика + AI-анализ вчерашнего дня + мотивирующая картинка |
| 06:00–22:00 | Практика-вопрос «о чём сейчас мысли?» с настроенной периодичностью (например, каждый час; через неделю — раз в 4 часа с другим вопросом). Ответ записывается, под ним кнопки самооценки |
| ~12:00 | Случайный пункт из списка «хочу» — предложение сделать дело на удовольствие |
| ~15:00 | Напоминание о практике + подбадривающая картинка + напоминание про 3 добрых дела |
| ~17:00 | Напоминание практики концентрации |
| ~20:00 | Гипноз на ночь: напоминание + заранее загруженное аудио |
| Вечер | Вопрос «какие 3 добрых дела сделала?» → ответ записывается |

Времена и состав — не хардкод, а конфигурация практик; таблица отражает стартовую настройку.

## Ограничения

- Один пользователь; whitelist по Telegram ID.
- Бюджет LLM/API: **$5–10/мес максимум** (расчётно ~$1–2/мес). Бюджетные модели
  (класс Haiku / gpt-mini). Подписки ChatGPT Plus / Claude Pro API-доступ не дают —
  нужен отдельный API-billing.
- Весь контент (напутствия, тексты/аудио практик, гипноз, картинки) загружается заранее
  вручную; AI контент практик не генерирует. Материалы уже существуют.
- Окно отправок: 06:00–22:00. Пропущенный чек-ин не «догоняется» повторными напоминаниями.
- Всё расписание — в локальном часовом поясе пользовательницы. Планируется год
  путешествий (Америка → Европа → Азия → Европа → Америка): смена часового пояса
  выполняется самостоятельно в боте, после смены всё расписание пересчитывается.
- Расписание одинаково все дни недели; есть кнопка «пропустить 1 день».
- Тон AI — только поддержка, без конфронтации. Диалог строго: спросил → записал →
  уточнил только при необходимости (если нет самооценки: «ведёт ли тебя это к твоим целям?»).
- Язык: русский; каркас i18n с первого дня (en/es добавляются без рефакторинга).
- Хостинг: существующий сервер, отдельный Docker-контейнер и отдельная БД.
- Этап 1 без веб-админки: контент и расписания настраивает оператор (конфиг/CLI/SQL).

## Не-цели

- Не групповой/мультипользовательский продукт (i18n — единственный задел на «пошарить»).
- Не свободный чат-коуч: бот не ведёт открытых бесед, не даёт непрошеных советов.
- Без генерации контента практик нейросетью.
- Без отдельной сущности «привычка» и без streak-статистики (привычка = практика).
- Без повторных напоминаний при пропуске сообщения.
- Просмотр дневника в самом боте не нужен (история — через веб, этап 2).
- Отчёты без графиков (текст); графики — отдельная проработка при необходимости.

## Критерии приёмки

### Этап 1 — MVP (бот)

- [ ] AC-1: Практика описывается данными (не кодом): название, тип контента
  (вопрос / текст / аудио / изображение), контент или ссылка на файл, периодичность
  (раз в N часов / в фиксированные времена), активность, даты начала/конца.
- [ ] AC-2: Планировщик отправляет активные практики точно по расписанию в окне 06:00–22:00;
  аудиофайлы (гипноз в 20:00) доставляются как загруженные, без изменений.
- [ ] AC-3: Утренний блок в 06:00 содержит: очередное напутствие из списка (ротация),
  утреннюю практику, AI-анализ вчерашнего дня, картинку из базы.
- [ ] AC-4: Изменение состава практик («+1 добавить, 1 убрать») и периодичности выполняется
  без изменения кода и без ручного перезапуска планировщика.
- [ ] AC-5: Команда/кнопка «пропустить день» отключает все отправки до конца текущего дня.
- [ ] AC-6: Любой ответ на сообщение бота сохраняется в дневник с привязкой к практике,
  дате и времени.
- [ ] AC-7: Голосовой ответ транскрибируется (Groq Whisper или аналог) и сохраняется
  текстом; исходный аудиофайл не хранится.
- [ ] AC-8: После записи мысли бот показывает кнопки самооценки
  «ведёт к целям ✅ / не ведёт ❌»; если оценка не поставлена — задаёт уточняющий вопрос
  «ведёт ли тебя это к твоим целям?» (детерминированная фраза, без LLM).
- [ ] AC-9: Список «хочу» пополняется командой в боте; в настроенное время (12:00) бот
  присылает случайный пункт.
- [ ] AC-10: Напоминание «3 добрых дела» днём; вечером бот спрашивает какие — записывает
  ответ в дневник.
- [ ] AC-11: Каждое утро AI-анализ за вчера: статистика самооценок («5 из 10 мыслей вели
  к целям»), поддерживающая формулировка, бюджетная модель; стоимость одного анализа
  ≤ $0.05.
- [ ] AC-12: Отчёт за произвольный период по запросу в боте (динамика самооценок,
  выполненные практики, добрые дела). Формат — текст, без графиков.
- [ ] AC-13: Тон всех AI-текстов — поддерживающий; промпт фиксирует запрет на критику
  и непрошеные советы.
- [ ] AC-14: Все строки интерфейса — через слой i18n (ru); добавление языка не требует
  правок логики.
- [ ] AC-15: Деплой: docker-compose на текущем сервере, отдельная БД, секреты через env.
- [ ] AC-16: Фактический расход API ≤ $10/мес при штатном использовании (минимум —
  логирование потраченных токенов).
- [ ] AC-17: Мотивирующие изображения отправляются 2 раза в день из заранее загруженной
  базы; повторы допустимы.
- [ ] AC-18: Часовой пояс меняется командой в боте (выбор: континент → страна → город);
  все расписания пересчитываются на новый локальный TZ без перезапуска.

### Этап 2 — Веб-админка

- [ ] AC-19: Веб-приложение с авторизацией через Telegram Login.
- [ ] AC-20: CRUD практик и расписаний; загрузка аудио/картинок/текстов.
- [ ] AC-21: Просмотр дневника и отчётов за период.
- [ ] AC-22: Управление списками «хочу» и напутствиями.

## Архитектурное решение

Отдельный проект (не доработка tg-smart-inbox, не agent-framework типа hermes-agent /
OpenClaw). Обоснование: домены с tg-smart-inbox не пересекаются; агентские фреймворки
прогоняют каждое cron-событие через полный LLM-цикл (~20 событий/день × большой системный
промпт — мимо бюджета при худшей предсказуемости), тогда как 19 из 20 событий здесь —
детерминированные отправки заранее загруженного контента.

**Стек:** Python 3.11+, aiogram 3.x, SQLAlchemy async + Alembic, APScheduler,
Anthropic API (Haiku-класс) или OpenAI gpt-mini-класс, Groq Whisper, Docker Compose.
Этап 2: FastAPI + Telegram Login Widget. Переиспользуются проверенные паттерны
tg-smart-inbox: слои Handler → Service → Repository, `transcription_service`, `i18n`,
`timezone_setup` FSM, `web/auth.py`.

### Оценка расходов (этап 1)

| Статья | Объём | Стоимость/мес |
|--------|-------|----------------|
| Утренний AI-анализ (Haiku-класс) | 1×/день, ~3–5k токенов вход | ~$0.2–0.5 |
| Транскрипция голоса (Groq whisper-large-v3-turbo, $0.04/ч) | ~30 коротких/день ≈ 7.5 ч/мес | ~$0.3 |
| Отчёты за период | по запросу | ~$0.1 |
| **Итого** | | **~$1, потолок бюджета $10** |

## Модель данных (онтология)

| Сущность | Тип | Поля | Связи |
|----------|-----|------|-------|
| User | core | telegram_id, timezone (меняется в боте), language | имеет все остальные сущности |
| Practice | **core, центральная** | name, content_type (question/text/audio/image), content/file_ref, periodicity, schedule_times, active, start/end | имеет много JournalEntry |
| JournalEntry | core | text (или транскрипт; исходное аудио не хранится), created_at, practice_id | принадлежит Practice; имеет SelfAssessment |
| SelfAssessment | core | leads_to_goals: bool, set_via (button/clarify) | принадлежит JournalEntry |
| MorningBlessing | supporting | text, порядок ротации | очередь ротации |
| MotivationalImage | supporting | file_ref | база с повторами |
| WantListItem | supporting | text, done? | случайный выбор в 12:00 |
| GoodDeed | supporting | text, date | вечерний опрос |
| DailyAIAnalysis | core | date, stats (n_of_m), message | агрегирует JournalEntry + SelfAssessment за день |
| PeriodReport | supporting | period, content (текст) | по запросу |
